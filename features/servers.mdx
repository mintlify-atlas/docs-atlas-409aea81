## Overview

Servers are community spaces where users can gather, communicate, and organize channels. Each server has an owner, members, and multiple channels for different topics or purposes.

## Server Model

The `Server` entity represents a community with the following structure:

```java
@Entity
public class Server {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(unique = true)
    private String name;

    private String imageUrl;

    @ManyToOne
    @JoinColumn(name = "owner_id")
    private User owner;

    @ManyToMany
    @JoinTable(
        name = "server_members",
        joinColumns = @JoinColumn(name = "server_id"),
        inverseJoinColumns = @JoinColumn(name = "user_id")
    )
    private Set<User> members;

    @OneToMany(mappedBy = "server", cascade = CascadeType.ALL, orphanRemoval = true)
    private List<Channel> channels;
}
```

<Info>
  Server names must be unique across the platform. The `members` field uses a `Set` to ensure users cannot join the same server multiple times.
</Info>

## Creating a Server

<Steps>
  <Step title="Submit server details">
    The user provides a server name and optional image URL.
  </Step>
  
  <Step title="User assigned as owner">
    The authenticated user automatically becomes the server owner and first member.
  </Step>
  
  <Step title="Default channel created">
    A "general" text channel is created automatically.
  </Step>
  
  <Step title="Server saved and returned">
    The server is persisted to the database and returned with full details.
  </Step>
</Steps>

### Create Server Implementation

```java
public ServerResponse createServer(ServerRequest request, Authentication connectedUser) {
    User user = (User) connectedUser.getPrincipal();

    Server newServer = Server.builder()
            .name(request.getName())
            .imageUrl(request.getImageUrl())
            .owner(user)
            .members(Set.of(user))  // Owner is automatically a member
            .build();

    // Create default "general" channel
    Channel defaultChannel = Channel.builder()
            .name("general")
            .type("TEXT")
            .server(newServer)
            .build();

    newServer.setChannels(List.of(defaultChannel));

    Server savedServer = serverRepository.save(newServer);
    return mapToResponse(savedServer);
}
```

<Note>
  The server creator is automatically added to the `members` set using `Set.of(user)`. This ensures "My Servers" queries work correctly from the moment of creation.
</Note>

## Joining a Server

Users can join existing servers to become members:

```http
POST /servers/{serverId}/join
Authorization: Bearer {jwt_token}
```

```java
public ServerResponse joinServer(Long serverId, Authentication connectedUser) {
    User user = (User) connectedUser.getPrincipal();

    Server server = serverRepository.findById(serverId)
            .orElseThrow(() -> new RuntimeException("Server not found"));

    // Add user to members (Set prevents duplicates)
    server.getMembers().add(user);

    Server savedServer = serverRepository.save(server);
    return mapToResponse(savedServer);
}
```

<Info>
  Because `members` is a `Set`, attempting to join a server you're already a member of has no effect.
</Info>

## Leaving a Server

Members can leave servers, but the owner cannot abandon their own server:

```http
POST /servers/{serverId}/leave
Authorization: Bearer {jwt_token}
```

```java
public void leaveServer(Long serverId, Authentication connectedUser) {
    User user = (User) connectedUser.getPrincipal();

    Server server = serverRepository.findById(serverId)
            .orElseThrow(() -> new RuntimeException("Server not found"));

    // Prevent owner from leaving
    if (server.getOwner().getId().equals(user.getId())) {
        throw new RuntimeException(
            "The owner cannot leave the server. You must delete it."
        );
    }

    // Remove user from members
    server.getMembers().removeIf(member -> member.getId().equals(user.getId()));

    serverRepository.save(server);
}
```

<Warning>
  Server owners cannot leave their own servers. They must delete the server instead, which will cascade delete all channels and content.
</Warning>

## Server-Channel Relationship

Servers have a one-to-many relationship with channels:

```java
@OneToMany(mappedBy = "server", cascade = CascadeType.ALL, orphanRemoval = true)
private List<Channel> channels;
```

<Note>
  The `cascade = CascadeType.ALL` and `orphanRemoval = true` settings ensure that when a server is deleted, all its channels are automatically deleted as well.
</Note>
