## Overview

Channels are communication spaces within servers. Each channel belongs to a specific server and has a type (TEXT, VOICE, or VIDEO) that determines how members interact.

## Channel Model

The `Channel` entity represents a communication channel:

```java
@Entity
@Table(name = "channels")
public class Channel {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    @Column(nullable = false)
    private String type = "TEXT";

    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "server_id", nullable = false)
    private Server server;

    @Column(name = "is_private")
    private Boolean isPrivate = false;
}
```

## Creating a Channel

<Steps>
  <Step title="Select parent server">
    Channels must be created within an existing server.
  </Step>
  
  <Step title="Provide channel details">
    Specify the channel name, type (TEXT/VOICE/VIDEO), and privacy setting.
  </Step>
  
  <Step title="Name formatting applied">
    Channel names are automatically converted to lowercase with spaces replaced by hyphens (Discord-style).
  </Step>
  
  <Step title="Channel added to server">
    The channel is associated with the server and saved via cascade.
  </Step>
</Steps>

### Channel Creation Implementation

```java
public ChannelResponse createChannel(Long serverId, ChannelRequest request) {
    // 1. Find the parent server
    Server server = serverRepository.findById(serverId)
            .orElseThrow(() -> new RuntimeException("Server not found"));

    // 2. Format channel name (lowercase with hyphens)
    String formattedName = request.getName()
            .trim()
            .toLowerCase()
            .replaceAll("\\s+", "-");

    // 3. Create the channel entity
    Channel newChannel = Channel.builder()
            .name(formattedName)
            .type(request.getType() != null ? request.getType() : "TEXT")
            .isPrivate(request.getIsPrivate())
            .server(server)
            .build();

    // 4. Add to server and save
    server.getChannels().add(newChannel);
    serverRepository.save(server);

    // 5. Return response
    return ChannelResponse.builder()
            .id(newChannel.getId())
            .name(newChannel.getName())
            .type(newChannel.getType())
            .build();
}
```

<Note>
  Channel names are automatically formatted: `"Tech Discussions"` becomes `"tech-discussions"`. This ensures consistency and URL-friendliness.
</Note>

## Channel-Server Relationship

Channels have a many-to-one relationship with servers:

```java
@ManyToOne(fetch = FetchType.LAZY)
@JoinColumn(name = "server_id", nullable = false)
private Server server;
```

<Info>
  Every channel must belong to a server. The relationship uses lazy fetching to optimize performance when loading channel lists.
</Info>

## Channel Mapping

The server service maps channel entities to DTOs:

```java
private ServerResponse mapToResponse(Server server) {
    List<ChannelResponse> channelResponses = server.getChannels() != null
            ? server.getChannels().stream()
                .map(channel -> ChannelResponse.builder()
                        .id(channel.getId())
                        .name(channel.getName())
                        .type(channel.getType())
                        .build())
                .collect(Collectors.toList())
            : List.of();

    return ServerResponse.builder()
            .id(server.getId())
            .name(server.getName())
            .imageUrl(server.getImageUrl())
            .channels(channelResponses)
            .members(memberResponses)
            .build();
}
```

## Cascade Deletion

Channels are automatically deleted when their parent server is deleted:

```java
@OneToMany(mappedBy = "server", cascade = CascadeType.ALL, orphanRemoval = true)
private List<Channel> channels;
```

<Warning>
  Deleting a server will permanently delete all associated channels and their messages. This action cannot be undone.
</Warning>
