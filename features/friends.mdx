## Overview

The friend system enables users to connect with each other through a two-step request workflow. Friends can see each other's online status and engage in direct messaging.

## Friendship Model

The `Friends` entity tracks relationships between users:

```java
@Entity
@Table(name = "friends")
public class Friends {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @ManyToOne
    @JoinColumn(name = "requester_id")
    private User requester;  // User who sent the request

    @ManyToOne
    @JoinColumn(name = "addresse_id")
    private User addressee;  // User who received the request

    @Enumerated(EnumType.STRING)
    private FriendShipStatus status;

    private LocalDateTime createdAt;
}
```

## Friendship Status

Friend relationships progress through different statuses:

```java
public enum FriendShipStatus {
    PENDING,   // Request sent, awaiting acceptance
    ACCEPTED,  // Friends - can see status and DM
    BLOCKED    // User blocked
}
```

## Friend Request Workflow

<Steps>
  <Step title="User sends friend request">
    The requester sends a friend request to another user by their ID.
  </Step>
  
  <Step title="Friendship created with PENDING status">
    A `Friends` record is created with status `PENDING`.
  </Step>
  
  <Step title="Addressee receives notification">
    The target user can view pending requests in their friend requests list.
  </Step>
  
  <Step title="Addressee accepts request">
    When accepted, the status changes to `ACCEPTED` and both users become friends.
  </Step>
</Steps>

## Sending Friend Requests

Users can send friend requests by specifying the target user's ID:

```http
POST /friends/request/{targetUserId}
Authorization: Bearer {jwt_token}
```

**Example:**

```http
POST /friends/request/42
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**Response:**

```json
{
  "message": "Friend request has been sent"
}
```

### Controller Implementation

```java
@PostMapping("/request/{targetUserId}")
public ResponseEntity<Map<String, String>> requestFriends(
        @PathVariable("targetUserId") Integer targetUserId,
        Authentication connectedUser
){
    friendsService.sendFriendRequest(targetUserId, connectedUser);
    return ResponseEntity.ok(Map.of("message", "Friend request has been sent"));
}
```

## Accepting Friend Requests

The addressee accepts a request using the friendship ID:

```http
PATCH /friends/accept/{friendshipId}
Authorization: Bearer {jwt_token}
```

**Example:**

```http
PATCH /friends/accept/15
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
```

**Response:**

```json
{
  "message": "Friend request has been accepted"
}
```

### Controller Implementation

```java
@PatchMapping("/accept/{friendshipId}")
public ResponseEntity<Map<String, String>> acceptFriendRequest(
        @PathVariable("friendshipId") Long friendshipId,
        Authentication connectedUser
){
    friendsService.acceptFriendRequest(friendshipId, connectedUser);
    return ResponseEntity.ok(Map.of("message","Friend request has been accepted"));
}
```

<Info>
  The friendship ID is returned in the pending requests list and must be used when accepting requests.
</Info>

## Status Updates

When a user logs in, their status is automatically set to ONLINE:

```java
public AuthenticationResponse authenticate(AuthenticationRequest request) {
    // ... authentication logic ...
    
    // Set status to ONLINE
    user.setStatus(UserStatus.ONLINE);
    userRepository.save(user);
    
    return AuthenticationResponse.builder()
            .token(jwtToken)
            .nickname(user.getNickName())
            .email(user.getEmail())
            .status(user.getStatus())
            .build();
}
```
